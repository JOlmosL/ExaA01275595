Preguntas a Responder

    ¿Por qué es una buena práctica separar el modelo del controlador? - Permite la detección rápida y eficiente de errores además de que nos permite dividir en multiples partes el código trabajado.

    ¿Qué es SQL injection y cómo se puede prevenir? - Es una vulnerabilidad que permite al atacante enviar o “inyectar” instrucciones SQL de forma maliciosa y malintencionada dentro del código SQL programado para la manipulación de bases de datos, haciendo que todos los datos almacenados corran peligro. Entre las diferentes acciones que se pueden realizar, se menciona:

    • Modificar el comportamiento de nuestras consultas a través de parámetros no deseados
    • Falsificar identidades
    • Obtener y divulgar información de la base de datos (contraseñas, correos, información relevante, entre otros)
    • Borrar la base de datos
    • Cambiar el nombre a las tablas
    • Anular transacciones
    • Etc... [1]
    Puede prevenirse haciendo uso de las siguientes técnicas:
  1. Convertir siempre el valor a su tipo correspondiente
  2. Parametrizar las consultas SQL
  3. Usar una cuenta con permisos restringidos a la base de datos
  4. No mostrar al usuario la información de error generada por la base de datos
  5. Rechazar las peticiones con caractéres sospechosos [2]
